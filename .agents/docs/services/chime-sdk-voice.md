# Amazon Chime SDK Voice

Source: AWS Smithy API model vendored in `vendor/api-models-aws`.

## Service Overview

The Amazon Chime SDK telephony APIs in this section enable developers to create PSTN calling solutions that use Amazon Chime SDK Voice Connectors, and Amazon Chime SDK SIP media applications. Developers can also order and manage phone numbers, create and manage Voice Connectors and SIP media applications, and run voice analytics.

## Official AWS Documentation Research

Sources:
- https://docs.aws.amazon.com/chime-sdk/latest/ag/voice-connectors.html
- https://docs.aws.amazon.com/chime-sdk/latest/dg/route-calls-events.html
- https://docs.aws.amazon.com/chime-sdk/latest/dg/mtgs-sdk-cvc.html

Research outcomes:
- Chime SDK Voice includes Voice Connectors, SIP media applications, SIP rules, phone numbers, logging, streaming, and PSTN audio integrations.
- Voice Connectors provide SIP trunking and can be grouped for fault tolerance.
- SIP rules route calls to SIP media applications or Voice Connectors based on phone number or request URI host name.
- PSTN audio routes call events to Lambda functions, with priority and failover behaviour.
- Voice Connector phone numbers can be assigned or unassigned; unassigned numbers can remain billable.
- SIP integration can connect external voice infrastructure to Chime SDK meetings through attendee join tokens and SIP URIs.

Parity implications:
- Model Voice Connectors, connector groups, SIP media applications, SIP rules, phone numbers, proxy settings, logs, streaming, and call legs separately.
- Call routing should evaluate SIP rules and Lambda/SIP target state.
- Phone-number assignment should be independent from Voice Connector deletion and billing state.
