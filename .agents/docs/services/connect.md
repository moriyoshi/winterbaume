## Service Overview

Amazon Connect actions Amazon Connect data types Amazon Connect is a cloud-based contact center solution that you use to set up and manage a customer contact center and provide reliable customer engagement at any scale. Amazon Connect provides metrics and real-time reporting that enable you to optimize contact routing. You can also resolve customer issues more efficiently by getting customers in touch with the appropriate agents. There are limits to the number of Amazon Connect resources that you can create. There are also limits to the number of requests that you can make per second.

## Official AWS Documentation Research

Sources:
- https://docs.aws.amazon.com/connect/latest/adminguide/amazon-connect-get-started.html
- https://docs.aws.amazon.com/connect/latest/adminguide/about-routing.html
- https://docs.aws.amazon.com/connect/latest/adminguide/connect-queues.html

Research outcomes:
- Amazon Connect instances host contact centre configuration such as users, queues, routing profiles, flows, phone numbers, prompts, and metrics.
- Contacts are routed through flows, queues, routing profiles, agent availability, channel concurrency, priorities, and delays.
- Standard queues hold contacts for eligible agents, while agent queues target a specific agent and have highest priority.
- Routing profiles associate agents with queues, priorities, delays, and channel concurrency settings.
- Contact flows define customer and agent experience through blocks and can integrate with services such as Lex and Lambda.
- Claimed phone numbers route inbound voice contacts into configured flows.
- Historical and real-time metrics are derived from contact and agent activity records.

Parity implications:
- Model instances, users, security profiles, queues, routing profiles, flows, prompts, phone numbers, contacts, agent status, and metrics separately.
- Contact routing should evaluate queue membership, priority, delay, agent availability, and concurrency rather than simply assigning first agent.
- Flow imports/exports and block limits should be validated as flow-specific state.
